Create navigational links within a website

There are several techniques to navigate within a website. The semantic tag for navigation is the <nav> tag. You can navigate through your web site by using relative addressing. For example we will user the unordered list tag. (<ul>)

  • Page to page within the same folder.
  • Page to page between folders.
  • Staying on the current page.
  • Going to a specific place on the current page (on page navigation)

Navigational menus are placed in your <header> semantic area. They may also be placed in your <footer> area as a sitemap. The nav tag like other semantic tags should have a <hx> tag. HOWEVER this header os optional. If you omit the header tag you will receive an document outliner error: Untitled Section. until you are show how to avoid this error message, it will be acceptable on your document.

CSS Coding Styles Summary

Browser Defaults (no css) You have no control over the styling
Inline This is for a single specific element (tag) instance. Your css will be specified in a parameter within the element. This css affects ONLY the element tag that contains the specification. style="rule"; [rule; ...] <p style="color:red; width:50px;"> ... <p>
Embedded The specification of css as embedded happens in your <head> area. This css affects all similar elements on the webpage where the css is coded. <style>
selector{rule; [rule; ...]}
[selector{rule; [rule; ...]}]
</style>		 <style>
p {color:red; width:50px;}
h1 {font-style:italic}
</style>
Link (stylesheet) This is the preferred method of doing css. you place your css in a separate physical file. This file can then be used on ANY webpage in your website. The link to this file will be done in your <head> area. The css in this file is similar to embedded EXCEPT there is no style tag. selector{rule; [rule; ...]}
[selector{rule; [rule; ...]}]		 p {color:red; width:50px;}
h1 {font-style:italic}
